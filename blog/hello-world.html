Hello Hexo!

发表于 2017-04-21 | 分类于 tips技巧

hexo是一个简洁而又高效的博客框架，支持Markdown语法，可以用来编写文章并生成HTML静态页面，结合部署到github服务器上管理和发布。

用 Hexo是一个比较方便的方法! 装完Node.js再几个命令就把本地博客框架搭好了，再配置一下Github 的仓库地址和添加生成的ssh key就可以发布文章。

在Github端配置一下CNAME，还可以解析到指定的域名，这样完成了一个有自己域名的博客啦:D

如果需要想要更改博客的版面风格，可以到hexo的主题页面上查看下载设置方式。需要查看关于hexo的信息可以浏览官方文档 ，使用过程中有问题还可以在官方的故障排除页面提问。

Quick Start

安装 Node.js

由于版本不断更新，最好的方法是到node.js官网下载*.tar.gz，然后解压配置安装。

或者通过hexo官方文档推荐的nvm方式

安装 Hexo

npm install hexo-cli -g

初始化工作

使用hexo server命令，Hexo会在本地启动一个HTTP服务，
这样就可以在本地打开预览效果了，地址为http://127.0.0.1:4000，
停止预览可以使用Ctrl+C终止掉进程。

部署到github

添加ssh key

使用ssh-keygen 生成 ssh key

ssh-keygen生成时会询问输入文件名，这个直接回车按默认的来，
密码按自己的需要设置一下。

这样默认会在本地的～/.ssh目录下生成id_rsa，id_rsa.pub两个文件，
id_rsa是私钥，id_rsa.pub是公钥。

把里面的内容拷贝到github的ssh and GPG keys中 新建的SSH KEY里。
